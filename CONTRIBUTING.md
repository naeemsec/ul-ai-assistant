# 🤝 Contributing to UL AI Assistant

First of all, thank you for your interest in contributing to **UL AI Assistant**! ❤️

We welcome bug fixes, UI improvements, new features, documentation updates, and performance enhancements. Every contribution helps make this project better.

---

# 📋 Before You Start

Please read these guidelines before contributing.

- Keep your code clean and well-organized.
- Follow the existing coding style.
- Test your changes before submitting.
- Avoid making unrelated changes in a single pull request.
- Write meaningful commit messages.

---

# 🚀 How to Contribute

## Step 1 — Fork the Repository

Click the **Fork** button at the top-right corner of this repository.

Give your repository name and This will create your own copy of the project.

---

## Step 2 — Clone Your Fork

Clone your fork to your local machine.

```bash
git clone https://github.com/naeemsec/ul-ai-assistant.git
```

---

## Step 3 — Create a New Branch

Never work directly on the `main` branch.

Create a new feature branch.

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b feature/pdf-chat-ui
git checkout -b feature/gpa-calculator
git checkout -b feature/improve-sidebar
git checkout -b bugfix/login-error
git checkout -b docs/update-readme
```

The prefixes (`feature/`, `bugfix/`, and `docs/`) are naming conventions used to keep branches organized. They are optional but recommended.

---

## Step 4 — Make Your Changes

Implement your feature or bug fix.

Examples:

- Improve UI
- Fix bugs
- Optimize performance
- Add documentation
- Improve accessibility
- Add new features

---

## Step 5 — Test Your Changes

Before submitting your contribution, make sure:

- The project builds successfully.
- No existing features are broken.
- Your new feature works correctly.

---

## Step 6 — Commit Your Changes

Use a clear commit message.

Example:

```bash
git commit -m "Add PDF Chat loading animation"
```

---

## Step 7 — Push Your Branch

```bash
git push origin feature/your-feature-name
```

---

## Step 8 — Open a Pull Request

Go to your GitHub repository and click:

**Compare & Pull Request**

Describe:

- What you changed
- Why you changed it
- Screenshots (if UI changes)

Then submit your Pull Request.

---

# ✅ Review Process

Every Pull Request is reviewed before being merged.

Your contribution may receive:

- Suggestions for improvement
- Requested changes
- Approval
- Merge into the main project

Please don't be discouraged if changes are requested. Code reviews are a normal part of software development.

---

# 📌 Contribution Ideas

You can contribute by:

- Bug fixes
- UI improvements
- Performance optimizations
- Documentation
- Accessibility improvements
- AI feature enhancements

- Smart Notes
- Quiz Generator

- Large unrelated changes
- Formatting-only commits
- Duplicate code
- Hardcoded API keys
- Breaking existing functionality
